SAGEBOOK 使用说明

一个纯静态、本地优先的小说创作平台：打开即写，内嵌 AI 写作技能，支持本地 Ollama 与云端大模型，并能通过云同步在手机与电脑间互通书稿。

一、SAGEBOOK 是什么

SAGEBOOK 是一套部署在网页上的写作工具集，不依赖后端服务器，所有书稿默认保存在你浏览器的本地存储（localStorage）中，完全私有。它通过调用你自己的 AI（本地 Ollama 或云端 API）提供续写、润色、审查等能力。

✍️ 沉浸写作
三栏式编辑器：章节树 / 写作区 / AI 面板，不离开页面即可完成构思到成稿。
🤖 内嵌 AI 技能
12 项写作技能 + 编辑器内 5 项审查/生成技能，全部真实调用你配置的模型。
💾 实时保存
每一次输入自动写入本地，刷新或关闭浏览器不丢稿。
🔒 本地隐私
书稿默认只存在本机浏览器；是否上云由你决定是否开启同步。
🔄 跨设备同步
通过 GitHub Gist / Gitee 把书稿加密托管，手机与电脑共享同一份。

二、三步开始

  1. 配置 AI：打开 设置，选择「本地 Ollama」或「云端大模型」并填写模型信息（推荐本地 Ollama，免费用、数据不出本机）。
  2. 进入编辑器：回到首页点「开始写作」，或书稿页左上角 📚 SAGEBOOK 回首页再进入。
  3. 动手写 + 调 AI：直接在中间写作区动笔；选中文字 → 右侧面板切到「💬 AI 助手」或「⚡ 技能」→ 让 AI 续写、润色或审查。
提示首次使用请务必先到设置页配置模型，否则 AI 相关按钮会提示「未配置 AI」。

五、设置页面

地址：settings.html。集中配置 AI 引擎与云同步。

1. AI 引擎配置

本地 Ollama（推荐）
  • 填写 Ollama 服务地址（默认 http://localhost:11434）与模型名（如 qwen2.5:14b）。
  • 点 🔄 扫描本地模型 自动列出本机已装模型；点「测试」验证连通性。
  • 数据不出本机，免费、隐私最佳。
云端大模型
  • 填写 API 地址（兼容 OpenAI 格式，如 https://api.openai.com/v1）、API Key、模型名。
  • 点「测试连接」验证（公网 HTTPS 页若填了 http:// 地址会被浏览器拦截，需改 https 或用本地模式）。

2. 一键本地启动器（解决混合内容限制）

重要当你通过公网地址（https://sagebool.github.io）访问时，浏览器出于安全策略禁止网页连接本地 http://localhost 的 Ollama（混合内容限制）。此时「扫描本地模型」会提示改用本地模式。

解决：双击项目里的 start-local.command（macOS）/ start-local.bat（Windows），它会自动起一个本地服务器并打开浏览器，让你以 http://localhost 访问本站，从而正常使用本地 Ollama。无需安装任何额外软件。

六、写作技能页（完整 12 项）

地址：writing-skills.html。一个独立的「输入文本 → 选技能 → 看结果」工作台，适合脱离编辑器单独处理片段、设定草稿。

用法：左侧粘贴/输入文本 → 点任一技能卡片 → 右侧出 AI 结果 → 「📋 复制结果」或「💾 下载 .txt」。

技能用途
🧹去 AI 味消除机器感与模板化表达，让文字更自然、有个人风格。
✨润色文学性打磨，提升流畅度与节奏，不改原意与语气。
🔍扩写增加感官细节、心理与氛围，篇幅扩至 1.5–2 倍，不水字数。
➡️续写顺承文风往下写 200–300 字，自然推进情节。
📋大纲生成从片段提炼章节大纲（卷/章标题 + 要点 + 可展开支线）。
⚔️矛盾冲突挖掘文本张力，给出 3–5 个加强冲突/反转的切入点。
🌍设定生成由片段反向推导世界观（地理/势力/力量体系/规则/历史）。
💬对话生成创作自然、有潜台词与观点冲突的对话（2–4 人）。
💡情节脑暴提供 5 个可行发展方向/爽点，含看点与风险。
👤人物小传生成角色档案：身份、性格、欲望恐惧、核心矛盾、弧光。
📊章节点评从风格/节奏/人物/悬念四维点评，给 3 条修改建议。
🌟金句提炼提炼或创作 3–5 句金句，并注明适用场景。

八、手机 / 电脑云同步

本地书稿默认只存浏览器。开启云同步后，书稿会被托管到 GitHub Gist 或 Gitee 仓库，凭 Token 在多端共享——不受「公网页 vs 本地页」来源隔离影响。

第一步 · 在设置页配置（电脑端）

  1. 打开 设置 → 「云同步（手机/电脑互通）」。
  2. 选择平台：GitHub Gist 或 Gitee 仓库。
  3. 填写对应 Token：
    • GitHub：到 Personal Access Tokens 生成，勾选 gist 权限。
    • Gitee：到 Gitee 个人设置生成私有令牌，并填写「仓库（owner/repo）+ 文件路径」。
  4. （GitHub）可留空 Gist ID，首次「☁ 同步」会自动创建。
  5. 可选开启「自动定时同步」，设定间隔（默认 10 分钟，后台静默上传）。

第二步 · 同步使用

操作在哪效果
☁ 同步（上传）主编辑器工具栏本地书稿 → 云端；首次建 Gist，之后更新。顶栏显示「上次同步 X 前」。
⬇ 拉取（下载）主编辑器工具栏云端书稿 → 本地（覆盖前会确认）。

第三步 · 在手机 / 另一台电脑

用同一浏览器打开本站 → 设置页填入相同的 Token → 点「⬇ 拉取」即可下载书稿继续写。之后各自「☁ 同步」即可互通。

注意「⬇ 拉取」会覆盖本地当前书稿。多端同时写时，先「☁ 同步」再在另一端点「⬇ 拉取」，避免互相覆盖。

九、常见问题

Q1：在公网页（sagebool.github.io）连不上本地 Ollama？

这是浏览器混合内容安全限制：HTTPS 页不能请求本地 HTTP 地址。解决：用项目里的 start-local.command / start-local.bat 起本地服务器，以 http://localhost 访问即可。

Q2：点技能 / 续写提示「未配置 AI」？

先到 设置 配置本地 Ollama 或云端 API Key 并测试通过，再回编辑器使用。

Q3：AI 调用失败 / 报错？

依次检查：① 本地 Ollama 是否在运行（ollama serve）；② 云端 Key 是否正确、额度是否充足；③ 网络是否畅通；④ 若在公网页用本地模型，参见 Q1。

Q4：我的书稿存在哪？私密吗？

默认只存在你浏览器的 localStorage，不上传任何服务器，完全私有。只有你主动开启「云同步」才会托管到 Gist/Gitee（由你的 Token 控制权限）。

Q5：换设备或换浏览器书稿看不见？

本地存储按「浏览器 + 域名」隔离，不会自动跨设备。请用第八节的云同步在设备间搬运。

Q6：误删了章节能恢复吗？

删除前有二次确认；若已确认删除，可尝试从「☁ 同步」的云端版本「⬇ 拉取」恢复（前提是之前同步过）。
